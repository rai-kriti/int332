# HomeController.java
```java
package com.example.studentportal.controller;

import com.example.studentportal.model.Feedback;
import com.example.studentportal.service.MessageService;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.PostMapping;

@Controller
public class HomeController {

    private final MessageService messageService;

    public HomeController(MessageService messageService) {
        this.messageService = messageService;
    }

    @GetMapping("/")
    public String home(Model model) {
        model.addAttribute("feedback", new Feedback());
        return "index";
    }

    @GetMapping("/about")
    public String about() {
        return "about";
    }

    @PostMapping("/submit")
    public String submit(@ModelAttribute Feedback feedback, Model model) {
        String finalMessage = messageService.buildThankYouMessage(
                feedback.getName(),
                feedback.getMessage()
        );
        model.addAttribute("finalMessage", finalMessage);
        return "result";
    }
}
```

---

- `Feedback` receives submitted form data

- `@Controller` marks MVC controller
- `Model` sends data to HTML page
- Mapping annotations bind URLs to methods

# Maven lifecycle in simple words
- `validate` checks project correctness
- `compile` compiles source code
- `test` runs test classes
- `package` creates JAR or WAR
- `install` copies artifact to local repository

# Most important Maven commands
```bash
mvn validate
mvn compile
mvn test
mvn package
mvn clean
mvn install
mvn spring-boot:run
```

---

# What `mvn validate` does
```bash
mvn validate
```
Why:
- Confirms POM structure is valid
- Checks project metadata early
- Good first command after writing `pom.xml`

---

# What `mvn compile` does
```bash
mvn compile
```
Why:
- Compiles only main Java source code
- Produces `.class` files in `target/classes`
- Detects syntax or import errors

---

# What `mvn test` does
```bash
mvn test
```
Why:
- Compiles test code
- Runs JUnit tests
- Ensures app configuration is still healthy

---

# What `mvn package` does
```bash
mvn package
```
Why:
- Executes compile and test first
- Creates final JAR inside `target`
- Ready for execution or sharing

---

# What `mvn clean` does
```bash
mvn clean
```
Why:
- Deletes old `target` folder
- Removes stale compiled output
- Useful before fresh rebuild

---

# What `mvn install` does
```bash
mvn install
```
Why:
- Builds and stores artifact locally
- Places JAR in local Maven repository
- Useful when other local projects depend on it

---

# What `mvn spring-boot:run` does
```bash
mvn spring-boot:run
```
Why:
- Starts application directly from source project
- No need to run built JAR manually
- Best for development and classroom demos

# Common error 1: mvn not recognized
Problem:
```text
'mvn' is not recognized as an internal or external command
```
Fix:
- Install Maven properly
- Add Maven `bin` to PATH
- Reopen terminal after PATH update

---

# Common error 2: Java version mismatch
Problem:
- Maven uses older Java
- Spring Boot build fails

Fix:
```bash
java -version
mvn -version
```
Make sure both point to Java 17+

---

# Common error 3: Port 8080 already used
Problem:
```text
Web server failed to start. Port 8080 was already in use.
```
Fix:
- Stop conflicting app
- Or change port in `application.properties`
```properties
server.port=9090
```

---

# Common error 4: package name mismatch
Problem:
- File path and package differ
- Java compilation fails

Fix:
- Ensure package line matches folders exactly
- Example:
```java
package com.example.studentportal.controller;
```

---

# Common error 5: template not found
Problem:
```text
Error resolving template [index]
```
Fix:
- Put HTML in `src/main/resources/templates`
- Return correct name from controller
- Example return `index`, not `index.html`
